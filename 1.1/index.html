Grundläggande HTML5

Doctype

Doctype deklareras för att instruera webläsaren vilken version av html som sidan är skriven i.

För HTML5 används deklarationen <!DOCTYPE html>

Teckenkod

För att webläsaren ska veta vilken typ av teckenkodning som sidan använder måste detta deklareras. UTF-8 innehåller de flesta tecken som finns.

För HTML5 används deklarationen <meta charset="utf-8">

Språk

Man deklarerar vilket språk som är det huvudsakliga språket för sidan för att t.ex sökmotorer ska få en bättre filtrering.

Deklarationen görs i html-taggen på följande sätt <html lang="sv">

Validering

Validering gör man för att hitta fel i koden som inte följer den rekommenderade standarden för HTML5. Det finns olika valideringsverktyg som till exempel https://validator.w3.org/ som vi använder oss av i kursen.

Hur är ett HTML-dokument uppbyggt

Ett HTML-dokument har en struktur som bygger på element som öppnar och stänger sektioner av koden vilket resulterar i vart information ska visas. Nedan är ett exempel på hur grundstrukturen ser ut.

Struktur

body

I body-elementet finns allt innehåll för sidan såsom text, bilder etc.

head

head-elementet innehåller metadata om html-dokumentet såsom charset, styles etc.

html

html-elementet är det som definerar hela dokumentet och innehåller hela koden.

title

Definerar titeln för dokumentet vilket sedan visas i broserns toolbar och i sökmotorer.

Exempel

<!DOCTYPE html>
<html>

  <head>
    <title>Title</title>
  </head>

  <body>
    <p>Hello World</p>
  </body>

</html>

Metainformation

author

Definerar författaren av en fil/sida.

Exempel: <meta name="author" content="Niclas Lövdahl">

description

Beskrivning av fil/sida.

Exempel: <meta name="description" content="Detta är en kurs i html.">

generator

Specifierar vilken mjukvara som använts för att skapa dokumentet.

Exempel: <meta name="generator" content="WordPress">

keywords

Specifierar en lista av nyckelord relevanta till sidan som sedan används av sökmotorer.

Exempel: <meta name="keywords" content="HTML, Tutorial">

robots

Meta robotstaggen används för att säga åt sökmotorer vilka sidor som ska visas eller döljas av sökmotorer. Det finns 4 huvudfunktioner FOLLOW, INDEX, NOFOLLOW och NOINDEX.

  • FOLLOW - Sökmotorn forsätter crawla länkarna på denna sida.
  • INDEX - Sökmoton indexerar sidan i fråga.
  • NOFOLLOW - Sökmotorn crawlar inte länkarna för sidan.
  • NOINDEX - Sökmotorn indexerar inte sidan i fråga.

Exempel: <meta name =”robots” content=”index”>

http-equiv

http-equiv används för att simulera en HTTP response header och kan användas om man till exempel inte har tillgång till server config filer.

Exempel: <meta http-equiv="refresh" content="60"> (Uppdaterar dokumentet varje minut)

Text

abbr

Taggen avnvänds för att beskriva eller skriva ut en förklaring av ett visst ord eller viss text. Man kan se denna beskrivning genom att hålla pekaren över ordet.

Exempel: <abbr title="HyperText Markup Language">HTML är kul <abbr title="Laughing Out Loud">LOL.

Resultat: HTML är kul LOL.

span

Span är en generisk container likt div men används istället i text. Det kan användas för att tex gruppera och/eller styla delar av text.

Exempel:

Hyptertext

Relativ och absolut länk

En relativ länk är en länk till ett dokument/fil som har en sökväg relativt till det dokumment som visas.

Exempel: <a href="/images/bild.png">Bildlänk</a>

En absolut länk specifierar hela sökvägen till länken, inklusive domän.

Exempel: <a href="https://www.google.se">Länk till Google</a>

Länk till ankare

En länk till en ankare är en länk som leder till en annan del i samma dokument. Man kan genom att referera till en annan tags id specifiera var i dokumentet man vill hamna.

Exempel: <a href="#top">Ta mig till toppen av dokumentet</a>

Resultat: Ta mig till toppen av dokumentet

Bild-länk

En bild-länk är en klickbar bild som fungerar som en länk.

Exempel: <a href="https://sv.wikipedia.org/wiki/Golden_retriever"> <img src="https://s-media-cache-ak0.pinimg.com/originals/91/d4/f0/91d4f0461db7abe765b90c7650d6e911.jpg" width="50%" height="50%"> </a>

Resultat:

Länk till olika protokoll

Man kan även skapa länkar som hänvisar till andra protokoll så som ftp (file transfer protocol).

Exempel: <a href="ftp://ftp.funet.fi/pub/standards/RFC/rfc959.txt">FTP Standard</a>

Resultat:

FTP Standard

Mailto-länk

Länk som tar en direkt till emailprogram på datorn med förbestämd mottagare.

Exempel: <a href="mailto:for.efternamn@example.com">Email me!</a>

Resultat:

Email me!
