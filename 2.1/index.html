Grunderna i CSS 3

Externt länkad CSS-fil och dess fördelar

Externt länkad CSS-fil innebär att man separerar CSS från HTML i en egen fil. Fördelarna med detta är att kan ha gemensamma stilregler för multipla filer och det kan underlätta om man har uppdelat arbete för CSS och html i ett team

Externt länkad CSS-fil inkluderas genom följande rad kod förutsatt att din CSS-fil har namnet style.css.

<link rel="stylesheet" type="text/css" href="style.css">

Arv

Arv i CSS fungerar så att CSS-regler till en specifik tagg ärvs ner till nästlade taggar. Till exempel en CSS-regler applicerade till <body>-taggen blir automatiskt applicerade till alla taggar på sidan. Alla taggar ärvs inte ner av specifika skäl och en tumregel är att taggar som hanterar placering inte ärvs. Till exempel så ärvs inte border ner eftersom det inte är sannolikt att man vill applicera det på alla element under.

Följande CSS-regel gör att all text på sidan kommer att bli blå.

body {
 color: blue;
}

Cascade

Cascade står för en vattenfallsmodell när det kommer till arv i CSS. Det finns olika regler för hur regler ärvs.

Ärvda regler slås ihop

Om regler ärvs från två olika taggar så slås dom automatiskt samman. Detta kan vanligtvis hända då man har nästlade taggar med flera förfäder.

Närmsta förfader vinner

Om en konflikt skulle uppstå där samma regel ärvs från två olika förfäder så gäller det att närmsta förfadern vinner.

Direkt applicerad regel vinner alltid

Eftersom direkt applicerad stil är närmre elementet än den närmsta förfadern så kommer den alltid att vinna vid konflikter.

En tagg, flera stilar

Om det inte uppstår några konflikter slås reglerna ihop precis som i multipla arv. Om det uppstår konflikter används "Specificity" för att bedöma vilken av reglerna som ska användas. Webbläsaren använder sig av ett poängsystem där selektorer i följd adderas och högst poäng vinner. Om poängen blir lika så används den senaste listade regeln i CSS-filen. Se poäng nedan.

  • Tagg-stil (specificity = 1)
  • Klass-stil (specificity = 10)
  • ID-stil (specificity = 100)

Selektorer

Varje CSS-stil består av selektor och deklarationsblocket. Selektorn bestämmer vad det är som ska appliceras stil på och deklarationsblocket bestämmer vad det är för stil. I exemplet nedan är selektorn i fet text och deklarationsblocket är hakparenteserna och allt innanför dessa.

body {
 color: blue;
}

Tagg-selektor

Tagg-selektorer används för att göra förändringar till specifika taggar på en sida och är användbart om man vill göra förändringar till flera element av samma tagg. Se exempel på tagg-selektor nedan.

p {
 font-size: 18px;
}

Attribut-selektorer

Attribut-selektorer gör det möjligt att ge stil till element som har blivit tilldelade ett specifikt attribut. Se exempel på attribut-selektor nedan.

input[type="button"] {
 color: red;
}

Klass-selektorer

Klass-selektorer är bra att använda då man ska styla ett specifikt element annorlunda från andra element på sidan. Man applicerar klass-selektorn genom att ange klassens namn i HTML-taggen. Klass-selektorer börjar med punk och måste börja med en bokstav efter punkten. Se exempel på klass-selektor nedan.

.classSelectorExample {
 font-family: "Times New Roman", Times, serif;
}

ID-selektorer

ID-selektorer fungerar på samma sätt som klass-selektorer och har inga fördelar förutom id kan användas av andra anledningar i JavaScript på en sida. Se exempel på ID-selektor nedan.

#idSelectorExample {
 color: black;
}

Barn-selektorer

Barn-selektorer innebär att man vill ge stil till ett element som är direkt barn till ett specifikt element. Regeln gäller alltså endast för barn som är direkt nästlade till det givna elementet. Se exempel på barn-element nedan.

body > p {
 color: pink;
}

Enheter

CSS använder sig av olika enheter för att ange en storlek på ett värde för en regel. Dessa består av absoluta och relativa enheter.

Absoluta enheter

  • in - inches
  • cm - centimeter
  • mm - millimeter
  • pt - punkter
  • pc - picas
  • px - pixlar

Relativa enheter

  • em - relativ till font-size av elementet
  • ex - relativ till x-height av den aktuella fonten
  • % - relativ till förfader-elementet

Hur ett CSS-dokument är uppbyggt

Ett CSS-dokument består oftast av flera CSS-regler. Som tidigare nämnt består dessa CSS-regler av en selektor och ett deklarationsblock. Hur CSS-reglerna fungerar har beskrivits i tidigare avsnitt. Se exempel på hur ett CSS-dokument kan se ut nedan.

body {
 color: pink;
}

body > p {
 color: yellow;
 font-family: "Times New Roman", Times, serif;
}

.box {
 padding: 25px 50px 75px 100px;
 border-style: solid;
 border-width: 5px;
}
